# 投票系统 - 宝塔面板部署教程

## 一、技术栈说明

| 技术 | 版本 | 说明 |
|------|------|------|
| Node.js | 20+ | 运行环境 |
| Next.js | 16 | 前端框架 |
| TypeScript | 5 | 编程语言 |
| PostgreSQL | - | 数据库（通过Supabase） |
| PM2 | - | 进程管理 |
| Nginx | - | 反向代理 |

---

## 二、准备工作

### 1. 购买服务器
- 推荐：2核4G内存以上
- 系统：Ubuntu 20.04 / 22.04 或 CentOS 7/8

### 2. 购买域名（可选）
- 需要备案（国内服务器）
- 配置DNS解析到服务器IP

### 3. 获取Supabase数据库信息
1. 访问 [Supabase](https://supabase.com) 注册账号
2. 创建新项目
3. 获取以下信息：
   - `DATABASE_URL`: 项目设置 → Database → Connection string (URI)
   - `SUPABASE_URL`: 项目设置 → API → Project URL
   - `SUPABASE_ANON_KEY`: 项目设置 → API → anon public key

## 三、安装宝塔面板

### Ubuntu/Debian:
```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484bec
```

### CentOS:
```bash
yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec
```

安装完成后会显示：
- 面板地址
- 用户名和密码

## 四、安装环境

登录宝塔面板后，安装以下软件：

### 必装软件：
1. **Nginx** - 任意版本
2. **PM2管理器** - 用于Node.js进程管理
3. **Node.js版本管理器** - 安装Node.js 20+

### 安装Node.js 20+：
1. 软件商店 → Node版本管理器 → 设置
2. 安装 Node.js v20 或更高版本
3. 设置为默认版本

### 方法一：宝塔文件管理
1. 创建目录：`/www/wwwroot/vote-system`
2. 将 `vote-system-source.tar.gz` 上传到此目录
3. 解压文件

cd /www/wwwroot

## 六、配置环境变量

在项目根目录创建 `.env` 文件：

```bash
cd /www/wwwroot/vote-system
nano .env
```

填入以下内容（替换为你的真实信息）：

```env
# 数据库配置（必填）
DATABASE_URL=postgresql://postgres:你的密码@db.你的项目ID.supabase.co:5432/postgres

# Supabase配置（必填）
SUPABASE_URL=https://你的项目ID.supabase.co
SUPABASE_ANON_KEY=你的anon_key

# 服务配置
PORT=5000
NODE_ENV=production

# 域名（可选）
COZE_PROJECT_DOMAIN_DEFAULT=你的域名
```

## 八、PM2启动服务

### 方法一：宝塔PM2管理器
1. 软件商店 → PM2管理器 → 设置
2. 添加项目：
   - 项目路径：`/www/wwwroot/vote-system`
   - 启动文件：`ecosystem.config.js` 或 `pnpm start`
   - 端口：5000

### 方法二：命令行
```bash
cd /www/wwwroot/vote-system

# 创建PM2配置文件
cat > ecosystem.config.js << 'EOF'
module.exports = {
  apps: [{
    name: 'vote-system',
    script: 'pnpm',
    args: 'start',
    cwd: '/www/wwwroot/vote-system',
    env: {
      NODE_ENV: 'production',
      PORT: 5000
    }
  }]
}
EOF

# 启动服务
pm2 start ecosystem.config.js

# 保存PM2配置（开机自启）
pm2 save
pm2 startup
```

### 验证服务：
```bash
pm2 status
curl http://localhost:5000
```

## 九、Nginx反向代理

### 方法一：宝塔面板配置
1. 网站 → 添加站点
   - 域名：你的域名（或IP）
   - 不创建数据库

2. 点击站点 → 反向代理 → 添加反向代理
   - 代理名称：vote-system
   - 目标URL：`http://127.0.0.1:5000`
   - 发送域名：`$host`

### 方法二：手动配置Nginx
```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 十三、常见问题

### 1. 服务启动失败
```bash
# 查看日志
pm2 logs vote-system

# 检查端口占用
netstat -tlnp | grep 5000
```

### 2. 数据库连接失败
- 检查 `.env` 中的数据库配置是否正确
- 确认Supabase项目未暂停
- 检查服务器是否能访问外网

### 3. 页面无法访问
```bash
# 检查服务状态
pm2 status

# 检查Nginx状态
systemctl status nginx

# 查看Nginx错误日志
tail -f /www/server/nginx/logs/error.log
```

### 4. 修改代码后重新部署
```bash
cd /www/wwwroot/vote-system
pnpm build
pm2 restart vote-system
```
